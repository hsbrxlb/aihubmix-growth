AIHubMix 第一批流量渠道

1. 搜索流量

流量从哪来

用户在 Google 搜 OpenRouter alternative、Claude cost reduction、AI API gateway、Dify model provider 这类问题。

放在哪里 / URL

主要放在 AIHubMix 自己的网站和文档站。

已有入口可以参考：

  • https://aihubmix.ai/
  • https://aihubmix.com/?lang=en
  • https://docs.aihubmix.com/en/index

新页面可以做成：

  • https://aihubmix.ai/openrouter-alternative
  • https://aihubmix.ai/claude-cost-reduction
  • https://docs.aihubmix.com/en/blogs/dify-model-provider
  • https://docs.aihubmix.com/en/blogs/ai-api-gateway-comparison
做什么内容

问题答案页、对比页、接入指南。不是首页大介绍，而是直接回答用户搜索的问题。

看什么结果

搜索进站后有没有注册、创建 API key、进入文档、完成首次调用。

2. 开发者社区流量

流量从哪来

开发者在 GitHub issue、工具论坛、Reddit、Discord 里问模型接入、成本、稳定性问题。

放在哪里 / URL

不是 AIHubMix 自己的网站，而是外部社区。比如：

  • GitHub issues：https://github.com/.../.../issues
  • Reddit：https://www.reddit.com/r/openrouter/、https://www.reddit.com/r/ChatGPTCoding/
  • Discord：相关 AI coding / AI tool 社区
  • 工具论坛：Dify Forum、LangChain Forum、LiteLLM GitHub 等
做什么内容

真实身份回答问题，给文档或示例链接。不是批量评论，也不是小号引流。

看什么结果

这些回答有没有带来文档访问、API key 创建、首次调用。

3. GitHub 流量

流量从哪来

开发者在 GitHub 搜示例项目、模板、SDK、provider、routing、OpenAI-compatible config。

放在哪里 / URL

AIHubMix 应该用自己的 GitHub repo 承接。现在能搜到一个相关 repo：

  • https://github.com/inferera/aihubmix

未来可以新增或整理成：

  • https://github.com/inferera/aihubmix/tree/main/examples/cline
  • https://github.com/inferera/aihubmix/tree/main/examples/dify
  • https://github.com/inferera/aihubmix/tree/main/examples/langchain

或者单独 repo：

  • https://github.com/inferera/aihubmix-starter-examples
做什么内容

最小可运行示例。不要一开始做大工程，先让用户能复制配置、填 API key、跑通一次。

看什么结果

repo 访问、star、fork、README 点击、从 GitHub 到文档/API key 的转化。

4. 技术内容流量

流量从哪来

开发者看教程、成本对比、接入指南、视频 walkthrough 时发现 AIHubMix。

放在哪里 / URL

优先放在 AIHubMix docs / blog：

  • https://docs.aihubmix.com/en/index

已有类似内容：

  • https://docs.aihubmix.com/en/blogs/free-ai-models

也可以放在外部平台：

  • Dev.to
  • Medium
  • Hashnode
  • YouTube
  • 技术博客
  • 合作方文档

新内容 URL 可以是：

  • https://docs.aihubmix.com/en/blogs/connect-cline-with-aihubmix
  • https://docs.aihubmix.com/en/blogs/dify-openai-compatible-provider
  • https://docs.aihubmix.com/en/blogs/langchain-model-routing
做什么内容

短教程、长文档、成本计算、迁移指南、5 分钟视频。

看什么结果

教程读完后有没有进入接入流程，有没有创建 API key。

5. 社媒流量

流量从哪来

开发者、AI 工具用户、独立开发者在 X、YouTube、LinkedIn、Indie Hackers 看到新工具信息、成本对比、案例和短教程。

放在哪里 / URL
  • X 官方账号
  • YouTube channel
  • LinkedIn company page
  • Indie Hackers profile / posts
  • 也可以做个人技术账号，但不要伪装小号。

这里 URL 要看 AIHubMix 现在有没有官方账号；如果没有，就应该建。HTML 可以先写类别，不要编具体账号 URL。

做什么内容

成本对比图、模型路由案例、产品更新、短教程、真实用户案例、工具接入视频。

看什么结果

不是看点赞，而是看有没有导到 docs、GitHub repo、API key 创建。

6. 生态 / KOL 流量

流量从哪来

开发者博主、AI coding 教程作者、开源项目维护者、工具集成伙伴把 AIHubMix 带进他们的教程、模板或项目里。

放在哪里 / URL

外部教程频道、GitHub 项目、AI SDK provider 页面、工具生态文档。

比如现在能看到 AIHubMix 已经出现在 Vercel AI SDK 社区 provider 生态里：

  • https://ai-sdk.dev/providers/community-providers/aihubmix

还可以争取出现在：

  • Dify 插件/模板生态
  • LangChain 示例
  • LiteLLM 相关对比
  • AI coding 博主教程
  • Open WebUI / LobeHub / Cherry Studio 等客户端接入文档
做什么内容

教程合作、工具模板、集成案例、真实用户故事。

看什么结果

外部引用、合作教程、集成使用、团队级用户。
