01 / Model Choice Playbook

模型选择手册

让开发者清楚：哪些任务需要高质量模型，哪些任务可以放心交给低成本、开源或免费模型。

当前卡点

开发者往往习惯把复杂任务、普通任务、批量任务都交给同一种高价模型，直到成本上来之后，才开始回头找替代方案。

具体动作

按任务场景拆分模型组合：关键任务保留高质量模型，高频但风险较低的任务，引导到 Qwen、DeepSeek、Gemini 或免费模型。

落地产物

任务矩阵、模型选择树、复制即用的配置、成本对比截图、短视频和 GitHub README 模板。

02 / Presets

一键配置方案

模型列表越长，用户越容易停在研究阶段。Preset 的价值，是提前替开发者做好选择，让复制配置比自己挨个比较模型更简单。

四个容易上手的接入包

围绕开发者已经在用的工具链，先做小而清楚的接入包，再用文章、视频和 GitHub 模板分发出去。

  • OpenCode 省钱 coding setup
  • Cline non-Claude agent setup
  • Dify support bot setup
  • LangChain fallback template
Balanced agent preset 
base_url: "https://aihubmix.com/v1"
route:
  coding: premium_when_needed
  summary: low_cost_default
  fallback: stable_paid_model
Low-cost batch preset 
tasks:
  logs: open_or_low_cost
  drafts: open_or_low_cost
  eval: compare_before_upgrade

03 / Onboarding

默认引导入口

第一次 API 调用会直接影响用户对产品的第一印象。新用户生成 key 后，应该直接看到三条能马上跑起来的路线，而不是只看到一个高价模型示例。

¥

省钱优先

默认使用低成本和免费模型，适合批量摘要、草稿、日志和分类任务。

≈

平衡优先

关键任务走高质量模型，高频任务走非 Claude 路线，并提前设置 fallback。

★

效果优先

保留 Claude / GPT 作为主链路，再通过看板找出后续可以迁移的任务。

04 / Cost Dashboard

成本看板

迁移的动力来自看得见的收益：让用户清楚看到 Claude 花了多少、非 Claude 花了多少、哪些任务可以迁移、迁移后大概能省出多少空间。

看板关注点 Weekly model-mix review
Claude spend 关键路径是否稳定
Non-Claude spend 高频任务是否在扩大

看板要回答的四个问题

  1. 哪些任务正在消耗高价模型？
  2. 哪些任务迁移后风险低？
  3. 迁移后效果是否够用？
  4. 本周非 Claude 调用占比有没有上升？

30 / 60 / 90

推进节奏

先证明开发者愿意复制配置，再把配置变成 onboarding 的默认路径，最后用数据持续调整模型组合。

第 1-2 周

冷启动材料

发布模型选择矩阵、4 个 starter preset、3 篇场景文章和 2 个短视频 demo。

第 3-6 周

工具链转化

把 Cline、OpenCode、Dify、LangChain 接入路径做成官网入口和 GitHub 模板。

第 7-12 周

模型组合复盘

上线成本看板、周报提醒和可迁移任务建议，把非 Claude 占比纳入运营指标。

Operating risks

需要提前守住的边界

免费模型的稳定性

免费入口适合拉新和试用，但需要说清楚额度、速度和可用性边界，避免用户把体验波动归咎于产品本身。

上游模型透明度

价格、provider、fallback 和路由逻辑越透明，欧美开发者越容易建立长期信任。

内容不能只讲便宜

核心表达应该是“不同任务匹配不同模型”，便宜只是结果之一。否则容易让人觉得 AIHubMix 只是一个低价接口转售平台。

指标先小后大

先看 first API call、preset copy、weekly active keys；跑顺之后，再看非 Claude 消耗占比和付费留存。
